## `claudex`

Launch the unified Claude Code session:

```powershell
claudex
```

This behaves the same with zero, one, or multiple connected providers. It never
opens a profile picker. With zero providers, responses come from the local
onboarding model until `/models` is used.

## `claudex models`

Open the loopback provider and model manager in the default browser:

```powershell
claudex models
claudex models open
```

The managed Claude Code skill invokes `claudex models open`, which is why
typing `/models` inside a session opens the same page.

## `claudex proxy`

```powershell
claudex proxy start
claudex proxy start --daemon
claudex proxy status
claudex proxy stop
```

The interactive bare command normally owns an embedded proxy that exits with
the session. A daemon is useful only when opening the model manager outside a
session.

## `claudex config`

Advanced configuration remains available for custom endpoints and existing
profile-based installations:

```powershell
claudex config show
claudex config path
claudex config validate
claudex config edit --global
```

## Legacy compatibility

The old `run`, `profile`, `dashboard`, and `auth` commands remain callable but
are hidden from normal help. They exist for existing custom-provider configs;
new installations should use bare `claudex` and `/models`.

Claude Free, Pro, and Max OAuth routing is rejected. Use an Anthropic Console
API key from `/models`.
