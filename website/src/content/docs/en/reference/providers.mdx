## Managed in-session accounts

| Provider | Protocol | Authentication | Model catalog |
|---|---|---|---|
| OpenAI | Responses API | ChatGPT browser OAuth | GPT-5.6 |
| Anthropic | Messages API | Console API key | Discovered from `/v1/models` for the key |

Both providers can be connected at the same time. Use `/models` to manage the
accounts and `/model` to switch between their models in one Claude Code session.

Claude Free, Pro, and Max OAuth credentials are not supported because
Anthropic's policy prohibits third-party gateways from offering or routing
those credentials. Anthropic access in Claudex uses a Console API key.

## Advanced endpoint adapters

The legacy configuration engine remains compatible with these endpoint types:

| Adapter | Examples |
|---|---|
| `DirectAnthropic` | Anthropic-compatible Messages APIs, MiniMax |
| `OpenAICompatible` | OpenRouter, Grok, DeepSeek, Kimi, GLM, Groq, Mistral, Together, Perplexity, Cerebras, Ollama, LM Studio, vLLM, Azure OpenAI |
| `OpenAIResponses` | OpenAI Responses-compatible gateways |

Custom endpoints are configured in `config.toml` and automatically join the
unified bare session. They are retained for compatibility and power-user use;
the normal OpenAI and Anthropic flow does not require profiles.

Provider calls are demand-driven. Starting Claudex or its proxy does not probe
any configured endpoint. Explicit account connection, connectivity tests, and
model requests are the operations that can contact providers.
