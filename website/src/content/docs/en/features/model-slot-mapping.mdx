---
title: Live Model Switching
description: Switch provider models with /model without restarting the agent
---

Joey's Claudex presents one combined picker and routes each exact model ID to
its connected provider.

```text
/model -> gpt-5.6-sol      -> OpenAI Responses API
/model -> claude-sonnet-5  -> Anthropic Messages API
```

## Live updates

When `/model` connects or removes an account, the running proxy rebuilds its
catalog in memory and the picker refreshes automatically. A session that
started with zero providers does not need to restart.

## Provider-neutral rows

The Claudex-owned picker displays the provider's exact model ID and provider
name. It does not synthesize custom Opus, Sonnet, or Haiku descriptions for GPT
models. The private onboarding identifier is never listed as a provider model.

## Ultracode effort

For a ChatGPT-backed GPT model, Claudex's effort selection is translated to
the OpenAI Responses API `reasoning.effort` field. The `ultracode` label maps to
`xhigh`. It also registers researcher, worker, and reviewer subagents that
inherit the active provider model and tells the main harness to delegate
independent work proactively and in parallel.

## Advanced routes

Existing `model_routes` configuration still works for custom endpoints and is
folded into the unified session. New account-first installations do not need to
write route tables manually.
