---
title: Provider Accounts
description: Persistent OpenAI and Anthropic accounts managed from /models
---

## Open the account manager

Run `/models` inside Claude Code. Claudex installs that command as a personal
skill before the session starts. If the user already owns a `/models` skill,
Claudex preserves it and installs `/claudex-models` instead.

The page is served by the loopback proxy and bundles all HTML, CSS, and
JavaScript in the Claudex binary. It makes no background provider request.

## OpenAI

Click **Connect OpenAI** to open ChatGPT browser authorization. Claudex uses
PKCE, validates OAuth state, binds the callback listener before opening the
authorization URL, and stores the resulting token in the OS credential store.
Required refreshes happen only when the selected OpenAI model is used.

## Anthropic API

Click **Connect Anthropic** to open Anthropic Console, create an API key, and
paste it into the localhost page. On that explicit action, Claudex validates the
key and requests the account's model catalog from Anthropic's Models API. The
key itself is stored in the OS credential store.

Claude Free, Pro, and Max OAuth credentials cannot be offered or routed by a
third-party gateway. Claudex rejects that legacy configuration and directs the
user to an Anthropic Console API key. See Anthropic's
[credential-use policy](https://code.claude.com/docs/en/legal-and-compliance).

## Persistent state

`~/.config/claudex/accounts.json` contains only provider labels, model IDs, a
credential-store reference, and the default model. It never contains a token or
API key. Adding or removing an account rebuilds the running model catalog in
memory, so `/model` can switch immediately without a new Claude Code session.
