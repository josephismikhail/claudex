## Install

<Tabs>
  <TabItem label="PowerShell" icon="terminal">
    ```powershell
    irm https://raw.githubusercontent.com/josephismikhail/claudex/main/install.ps1 | iex
    ```

    The Windows x64 installer verifies the release SHA-256 checksum, installs
    under `%LOCALAPPDATA%\Programs\Claudex\bin`, and adds that directory to the
    current user's `PATH`. WSL is not required.
  </TabItem>
  <TabItem label="Linux / macOS" icon="terminal">
    ```bash
    curl -fsSL https://raw.githubusercontent.com/josephismikhail/claudex/main/install.sh | bash
    ```
  </TabItem>
  <TabItem label="Cargo" icon="seti:rust">
    ```bash
    cargo install --git https://github.com/josephismikhail/claudex
    ```
  </TabItem>
  <TabItem label="GitHub Releases" icon="github">
    Download a signed release asset and its SHA-256 file from
    [GitHub Releases](https://github.com/josephismikhail/claudex/releases).
  </TabItem>
</Tabs>

## Requirements

- Windows x64, macOS, or Linux
- Claude Code installed as `claude` in `PATH`

## Start

```powershell
claudex
```

There is no initial configuration step. Joey's Claudex opens with an empty
provider catalog and `/model to authenticate` in its header:

1. Run `/model`.
2. Choose OpenAI or Anthropic.
3. Finish authentication in the browser and select a returned model.

The account, model catalog, and default model persist the next time you run
`claudex`.
