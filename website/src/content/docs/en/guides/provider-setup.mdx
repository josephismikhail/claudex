---
title: Provider Setup
description: Connect OpenAI and Anthropic from the local account manager
---

## Open the account manager

```powershell
claudex models open
```

A new installation deliberately has no preset provider models. Claudex does
not install a duplicate `/models` command; `/model` remains Claude Code's one
in-session model picker.

## OpenAI

1. Click **Connect OpenAI**.
2. Complete ChatGPT sign-in in the browser window.
3. Return to the account page and wait for **Connected**.
4. Use the copy button and run `/model gpt-5.6` immediately. On later
   launches, select it from the bare `/model` picker.

The OAuth callback stays on localhost, uses PKCE and state validation, and
stores the token in the operating system credential store.

## Anthropic API

1. Click **Connect Anthropic**.
2. Create an API key in the Anthropic Console tab that opens.
3. Paste the key into the Claudex localhost dialog.
4. Copy `/model <model-id>` for an immediate switch. Persisted models appear
   in the bare `/model` picker on later launches.

Claudex contacts Anthropic's Models API only as part of this explicit connect
operation. It does not probe the provider at startup.

Anthropic's credential policy prohibits third-party products from routing
Claude Free, Pro, or Max credentials. Use a Console API key here; keep Claude
subscription login inside official Claude products.

## Disconnect

Click **Remove** on a connected account. Claudex removes the metadata and asks
the operating system credential store to delete the corresponding secret. The
running model catalog updates immediately.

## Custom and local providers

The previous profile engine remains available for Ollama, LM Studio, vLLM,
OpenRouter, and other custom endpoints. It is an advanced compatibility path;
see `config.example.toml` in the repository.
