## Normal setup needs no config editing

Run `claudex`, then `/models`. Provider accounts and their models are managed by
the localhost page and persisted independently of project configuration.

Account metadata lives at:

```text
~/.config/claudex/accounts.json
```

Secrets do not live there. OpenAI tokens and Anthropic API keys use the
operating system credential store.

## Default model

Use **Make default** beside a connected model in `/models`. The choice applies
to future sessions. In a running Claude Code session, use `/model` to switch
without changing the persistent default.

## Base settings

The global compatibility config remains at
`~/.config/claudex/config.toml`:

```toml
proxy_port = 13456
proxy_host = "127.0.0.1"
log_level = "info"
hyperlinks = "auto"
```

The account manager is available only when `proxy_host` is loopback.

## Advanced legacy providers

Custom local and OpenAI-compatible endpoints can still be defined as profiles.
They automatically appear in the unified bare session, but profile commands are
hidden from normal help so they do not complicate first-run setup.

```toml
[[profiles]]
name = "local-ollama"
provider_type = "OpenAICompatible"
base_url = "http://127.0.0.1:11434/v1"
api_key = "ollama"
default_model = "qwen3"
```

See the repository's
[complete example](https://github.com/josephismikhail/claudex/blob/main/config.example.toml)
for router, fallback, context-engine, and custom-header fields.

Claude subscription OAuth profiles are unsupported and rejected. Anthropic
must be connected with a Console API key from `/models`.
